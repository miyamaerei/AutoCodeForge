多 Agent 流水线系统 —— 大屏需求 + 前端实现（最终精简版）
一、业务需求（仅大屏需要展示的核心逻辑）
7 步固定工序流水线
需求梳理 → 信息查询 → 方案制定 → 代码开发 → 测试校验 → 版本提交 → 最终审核
3 类 Agent
秘书（调度接单）、管理者（审核）、工人（执行）
Agent 4 种状态
Idle 空闲、Handling 处理中、Learning 学习中、Dormant 休眠
任务流转规则
任务按工序顺序流动
审核通过 → 进入下一步
审核驳回 → 退回上一步
超时 / 异常 → 任务标记失败
Agent 空闲自动学习，高优任务打断学习
大屏必须展示的信息
任务当前位置、状态、进度、归属 Agent
Agent 实时状态、工作负载、所在工位
系统整体运行统计
任务日志、Agent 日志、系统事件日志（实时滚动）
二、大屏前端功能（只写大屏！不写其他页面）
1. 画面形态：2D 仿真沙盘（类 AnyLogic / 游戏引擎风格）
中间：7 个工序工位 + 轨道动线
任务图标沿轨道平滑移动
Agent 图标显示在对应工位旁
状态用颜色 + 动画直观展示
2. 大屏必须具备的功能
任务实时流动画
移动 → 工位作业 → 完成移动 → 驳回退回
Agent 状态实时展示
空闲 / 工作 / 学习 / 休眠 可视化
系统实时统计面板
任务总数、运行中、异常数、完成数
各角色 Agent 状态分布
实时日志区域（底部滚动）
任务日志：创建、开始、完成、驳回、异常
Agent 日志：状态切换、开始工作、进入学习、休眠
系统日志：告警、超时、错误、调度
点击交互
点击任务 → 显示该任务完整日志
点击 Agent → 显示该 Agent完整日志
异常高亮
任务失败、超时、驳回、Agent 休眠 → 红色闪烁提示
3. 前端技术栈
Vue3 + Vue-Konva（Canvas 2D 沙盘）
三、后端接口（大屏专用 4 个，含你要的日志）
1. 系统快照（概览数据）
GET /api/dashboard/snapshot返回：系统状态、任务统计、Agent 统计
2. 实时任务列表（动线核心）
GET /api/dashboard/tasks返回：所有任务的当前工序、状态、进度、所属 Agent
3. 实时 Agent 列表
GET /api/dashboard/agents返回：所有 Agent 的角色、状态、所在工位、当前任务
4. 日志接口（你新增的关键接口！）
GET /api/dashboard/logs支持参数：
type: task / agent / system
taskId (可选)
agentId (可选)
返回格式：
json
[
  {
    "time": "2025-12-19 12:34:56",
    "type": "task",
    "taskId": "task-001",
    "agentId": "worker-01",
    "content": "任务进入【代码开发】工序",
    "level": "info"
  }
]
四、前后端交互规则（最终版）
前端 每 2 秒轮询
任务 + Agent + 快照
日志 每 1 秒轮询（或滚动加载）
后端只返回数据状态
不返回坐标、不计算动画
前端负责：
动画
动线移动
日志渲染
状态图标的切换
