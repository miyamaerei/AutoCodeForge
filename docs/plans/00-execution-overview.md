## 🏗️ 复用架构总览

本项目采用**分层复用架构**，从上到下依次复用：

```
┌─────────────────────────────────────────────────────────────┐
│                    业务层（阶段三~十）                       │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │ AI 核心  │  │ 任务中心  │  │ Git 集成 │  │  其他...  │   │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘  └────┬─────┘   │
└───────┼─────────────┼─────────────┼─────────────┼──────────┘
        │             │             │             │
        └─────────────┴─────────────┴─────────────┘
                      │
         ┌────────────┴────────────┐
         │   数据层 & 认证（阶段二）  │
         │  16+ 实体 / 8 个 Repository │
         └────────────┬────────────┘
                      │
         ┌────────────┴────────────┐
         │  基础设施层（阶段一）     │
         │  BaseRepository / 中间件 │
         │  Helper / 基类实体       │
         └─────────────────────────┘
```

### 复用收益总览

通过复用架构，整个项目预计可以**避免 5000+ 行重复代码**：

| 复用类别 | 避免重复代码 | 说明 |
|---------|-----------|------|
| 基类实体复用 | 2000+ 行 | 16+ 实体无需重复写时间戳、软删除、NtId |
| BaseRepository 复用 | 1500+ 行 | 12+ Repository 无需重复写 CRUD/分页 |
| ApiResponse 复用 | 500+ 行 | 30+ API 端点统一响应格式 |
| ExceptionMiddleware 复用 | 500+ 行 | 所有端点无需重复写异常处理 |
| Helper 工具复用 | 500+ 行 | 全局分页、JSON、时间处理工具 |

## 📐 任务拆分原则

每个任务都遵循以下拆分标准：

| 原则 | 说明 |
|------|------|
| **单一职责** | 每个任务只做一件事 |
| **可验证** | 每个任务都有明确的完成标准 |
| **可复用** | 优先创建可复用的基础设施 |
| **小步快跑** | 单个任务不超过 2 小时工作量 |
| **依赖清晰** | 明确标注前置依赖任务 |
