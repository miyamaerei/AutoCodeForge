# AutoCodeForge 用户使用指引

## 1. 系统概述

欢迎使用 **AutoCodeForge** - AI 驱动的代码自动化平台。本系统帮助您通过自然语言描述需求，自动生成代码并管理开发任务。

## 2. 快速入门

### 2.1 登录系统

1. 打开浏览器访问系统入口
2. 系统支持免登录体验模式
3. 直接进入控制台首页

### 2.2 系统首页

首页展示各功能模块入口卡片：

| 模块 | 功能描述 |
|------|----------|
| AI任务中心 | 自然语言提需求、多轮对话、执行步骤 |
| Agent中心 | 配置AI Agent，支持自动选择和手动引用 |
| 定时任务 | 配置自动化定时任务，支持仓库关联和模板 |
| 项目/仓库管理 | 仓库列表、分支、文件树与PR视图 |
| Dashboard工作台 | 今日任务数、成功率、最近任务与告警 |
| 流水线中心 | 构建状态、步骤状态、日志查看 |
| 系统配置 | API Key、模型选择、用户管理 |

## 3. 核心功能使用指南

### 3.1 AI任务中心（核心功能）

**场景**：通过自然语言描述需求，创建开发任务

#### 步骤1：进入任务中心
1. 点击左侧导航栏「AI任务中心」
2. 或点击首页「AI任务中心」卡片

#### 步骤2：创建新任务
1. 点击「+ 创建任务」按钮
2. 填写任务信息：
   - **任务标题**：简洁描述需求
   - **任务描述**：详细说明需求（支持自然语言）
   - **目标仓库**：选择代码仓库
   - **目标分支**：选择目标分支
3. 点击「提交」按钮

#### 步骤3：查看任务列表
- 任务列表展示所有任务
- 支持按状态筛选（运行中、已完成、已暂停、失败）
- 点击任务行可查看详情

#### 步骤4：查看任务详情
1. 点击任务进入详情页
2. 查看任务执行步骤
3. 查看任务日志
4. 查看代码变更（Diff）
5. 与AI对话讨论任务

**任务状态说明**：

| 状态 | 说明 |
|------|------|
| 运行中 | 任务正在执行 |
| 已完成 | 任务成功完成 |
| 已暂停 | 任务暂停等待 |
| 失败 | 任务执行失败 |

#### 使用技巧

- **快捷问题**：点击建议问题快速提问

### 3.3 Agent 中心

**场景**：配置 AI Agent，实现自动选择和手动引用

#### 查看 Agent 列表
1. 点击左侧导航「Agent」
2. 查看所有已配置的 Agent
3. 显示 Agent 名称、描述、启用状态

#### 创建新 Agent
1. 点击「+ 创建 Agent」按钮
2. 填写 Agent 信息：
   - **名称**：Agent 标识名称
   - **描述**：Agent 功能描述
   - **头像**：选择头像图标
   - **系统提示词**：定义 Agent 行为和角色
3. 配置自动选择关键词（可选）：
   - 添加关键词和权重
   - 如：「前端」权重 1.0、「React」权重 0.8
4. 点击「保存」

#### 编辑/删除 Agent
1. 点击 Agent 行右侧的「编辑」按钮
2. 修改信息后保存
3. 或点击「删除」按钮移除 Agent

#### Agent 自动选择机制
- 在聊天时，系统会根据消息内容自动匹配最合适的 Agent
- 匹配规则：消息中包含的关键词权重之和
- 只有匹配度超过阈值（0.5）才会自动选择
- 如果没有匹配的 Agent，系统会使用默认设置

#### 手动选择 Agent
1. 在聊天界面选择「指定 Agent」
2. 从下拉列表选择需要的 Agent
3. 后续对话将使用选定的 Agent

### 3.6 定时任务

**场景**：配置和管理自动化定时任务，支持仓库关联和任务模板

#### 查看定时任务列表
1. 点击左侧导航「定时任务」
2. 查看所有定时任务
3. 显示任务名称、关联 Agent、仓库、触发方式、状态

#### 创建新定时任务
1. 点击「+ 创建任务」按钮
2. （可选）从任务模板快速创建：
   - 代码审查任务：每天 9:00 执行代码审查
   - 数据库备份任务：每 6 小时执行备份
   - 文档生成任务：每周五 18:00 生成文档
   - 系统监控任务：每小时监控系统状态
   - 代码重构任务：一次性执行重构
3. 填写任务信息：
   - **任务名称**：简洁的任务标识
   - **任务描述**：描述任务职责
   - **关联仓库**：选择代码仓库（必填）
   - **目标分支**：如 main、develop
   - **目标路径**：如 src/utils（可选，留空表示整个仓库）
   - **关联 Agent**：选择执行任务的 Agent
4. 配置触发方式：
   - **Cron 表达式**：如 `0 9 * * *` 每天 9:00
   - **固定间隔**：如每 6 小时、每 30 分钟
   - **一次性**：指定具体时间执行一次
5. 配置任务参数（JSON 格式）
6. 点击「保存」

#### 管理定时任务
- **编辑**：点击任务行右侧「编辑」按钮
- **删除**：点击「删除」按钮
- **启用/禁用**：点击开关切换
- **立即执行**：点击「立即执行」按钮手动触发

#### 查看执行记录
1. 点击任务行查看详情
2. 查看「执行记录」标签
3. 显示每次执行的时间、耗时、状态、结果

**触发方式说明**：

| 触发类型 | 说明 | 示例 |
|---------|------|------|
| Cron 表达式 | 定时循环执行 | `0 9 * * *` 每天 9:00 |
| 固定间隔 | 按固定时间间隔执行 | 每 6 小时 |
| 一次性 | 指定时间执行一次 | 2026-06-01 10:00 |

**任务状态说明**：

| 状态 | 说明 |
|------|------|
| 空闲 | 等待下次执行 |
| 运行中 | 正在执行 |
| 失败 | 执行出错 |
| 禁用 | 已停止 |

### 3.7 系统配置

**场景**：配置系统参数和个人偏好

#### 3.7.1 仓库配置（重要！）

在使用任务中心等功能前，**必须先配置代码仓库**：

1. 进入「系统配置」→「Repositories」
2. 配置代码托管平台：

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| Provider | 代码托管平台 | GitHub / Azure DevOps / GitLab |
| Owner | 仓库所有者/组织 | `username` 或 `org-name` |
| Repository | 仓库名称 | `my-project` |
| 默认分支 | 主分支名称 | `main` / `master` |
| 鉴权方式 | 认证方式 | Token / App / OAuth |
| 合并策略 | PR合并方式 | Merge / Squash / Rebase |
| 检查门禁 | 是否启用CI检查 | 启用/关闭 |

3. 填写完成后点击「保存配置」
4. 系统会自动验证仓库连接

**支持的平台**：
- ✅ GitHub
- ✅ Azure DevOps
- ✅ GitLab

#### 3.7.2 定时任务（简版）

配置自动化定时任务（系统配置入口）：

1. 进入「系统配置」→「Schedules」
2. 配置任务参数：

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| 任务名称 | 定时任务标识 | `daily-knowledge-sync` |
| Cron | 执行时间表达式 | `0 2 * * *`（每天凌晨2点） |
| 时区 | 时区设置 | `Asia/Shanghai` |
| 失败重试 | 失败时重试次数 | 0-10次 |
| 启用状态 | 是否启用 | 启用/暂停 |
| 告警通道 | 通知方式 | In-App / Email / Webhook |

**常见Cron表达式**：
- `0 2 * * *` - 每天凌晨2点
- `0 */6 * * *` - 每6小时
- `0 9 * * 1-5` - 工作日上午9点
- `30 4 * * 0` - 每周日凌晨4:30

#### 3.7.3 知识库配置

配置AI知识来源：

1. 进入「系统配置」→「Knowledge」
2. 配置知识源参数：

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| 来源名称 | 知识源标识 | `project-docs` |
| 来源类型 | 文档格式 | Markdown / Remote Wiki / Repository |
| 来源路径 | 文档位置 | `/docs` 或 URL |
| 刷新策略 | 更新频率 | Manual / Hourly / Daily |
| Chunk Size | 分块大小 | 100-4000字符 |
| Overlap | 块重叠 | 0-1000字符 |
| 访问范围 | 访问权限 | Public / Internal / Restricted |

3. 配置完成后点击「保存配置」

**作用**：
- 为AI提供项目文档上下文
- 帮助AI更准确地理解项目
- 支持文档自动同步更新

#### 3.7.4 DeepWiki 配置

配置高级知识检索：

1. 进入「系统配置」→「DeepWiki」
2. 配置向量搜索参数：

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| Workspace | 工作区名称 | `AutoCodeForge` |
| Index Name | 索引标识 | `autocodeforge-main-index` |
| Embedding | 向量模型 | text-embedding-3-large / small |
| Top K | 返回结果数 | 1-50 |
| Metric | 距离度量方式 | Cosine / Dot Product / Euclidean |
| Retention Days | 数据保留天数 | 7-365天 |
| Auto Reindex | 自动重建索引 | 开启/关闭 |

3. 配置完成后点击「保存配置」

**作用**：
- 支持语义搜索
- 提供更智能的知识检索
- 基于向量相似度的文档匹配

#### 3.7.5 其他配置项

| 配置项 | 说明 |
|--------|------|
| Preferences | 个人偏好设置 |
| Skill | 自定义Skill管理 |
| DeepWiki | 向量知识库配置 |
| Review | 代码审查配置 |
| Integrations | 第三方服务集成 |
| Notifications | 通知推送设置 |
| Sandbox | 沙箱环境配置 |
| Workflow | 工作流自动化配置 |
| API | API访问配置 |
| Models | AI模型选择 |
| Users | 用户和权限管理 |

#### 3.7.6 第三方集成

**Azure DevOps 集成**：
1. 进入「系统配置」→「Integrations」
2. 选择「Azure PWT」
3. 配置认证方式（Microsoft Entra 或 PAT）
4. 填写 Tenant ID、Subscription ID 等信息
5. 保存配置

**GitHub Copilot 集成**：
1. 进入「系统配置」→「Integrations」
2. 选择「GitHub Copilot」
3. 配置认证方式（Interactive 或 PAT）
4. 可复制安装命令到本地执行
5. 配置 Workspace Policy（Required/Optional/Disabled）

## 4. Wiki系统

AutoCodeForge 提供两套Wiki系统，适用于不同场景：

### 4.1 MD Wiki（Markdown Wiki）

**适用场景**：基于Git仓库的Markdown文档系统

#### 使用步骤：
1. 在控制台点击「Wiki」或直接访问 `/mdwiki`
2. 选择仓库和分支
3. 浏览Markdown文档列表
4. 点击查看文档详情

#### 功能特点：
- ✅ 从Git仓库自动加载Markdown文档
- ✅ 支持文档目录和分类
- ✅ 渲染Markdown为HTML
- ✅ 自动生成目录导航
- ✅ 支持多仓库切换

### 4.2 Console Wiki

**适用场景**：项目知识管理和文档导航

#### 使用步骤：
1. 进入控制台「Wiki」页面
2. 从下拉框选择仓库
3. 左侧显示Wiki页面列表
4. 点击查看页面详情

#### 页面信息：
- 页面目的（Purpose）
- 页面笔记（Notes）
- 页面元信息（Meta）
- 作者、更新时间等

#### 功能特点：
- ✅ 结构化的Wiki元数据
- ✅ 页面分类和分组
- ✅ 笔记关联功能
- ✅ 目录导航（TOC）

### 4.3 Wiki配置

在系统配置中配置Wiki知识源：

1. 进入「系统配置」→「Knowledge」
2. 添加Wiki知识源
3. 设置来源类型为「Remote Wiki」或「Repository」
4. 配置文档路径
5. 设置刷新策略

## 5. 典型使用场景（完整流程）

### 5.1 首次使用准备工作

> ⚠️ **重要提示**：首次使用前，请先完成以下配置：

```
第一步：配置代码仓库
├─ 进入「系统配置」→「Repositories」
├─ 选择代码托管平台（GitHub/GitLab/Azure DevOps）
├─ 填写Owner和仓库名
├─ 设置默认分支（如main）
├─ 选择鉴权方式（推荐Token）
└─ 点击「保存配置」

第二步：配置知识库（推荐）
├─ 进入「系统配置」→「Knowledge」
├─ 添加知识源（项目文档、API文档等）
├─ 设置文档路径
└─ 配置刷新策略

第三步：选择AI模型（可选）
├─ 进入「系统配置」→「Models」
└─ 选择合适的AI模型

完成后即可开始使用！
```

### 5.2 场景一：修复代码Bug（推荐入门）

```
完整流程：
1. 进入「系统配置」→「Repositories」
   └─ 配置你的GitHub/GitLab仓库
   └─ 设置Owner、仓库名、分支
   └─ 选择鉴权方式（推荐Token）

2. 进入「AI任务中心」
3. 点击「创建任务」
   └─ 标题："修复订单导出空指针异常"
   └─ 描述："订单导出功能在某些情况下抛出空指针异常，请修复"
   └─ 仓库：选择已配置的仓库
   └─ 分支：选择目标分支（如main）

4. 点击「提交」
5. 系统开始执行任务
6. 等待AI分析和修复
7. 进入任务详情查看结果
   └─ 查看执行步骤
   └─ 查看代码变更（Diff）
   └─ 查看执行日志

8. 如有疑问，通过任务内聊天与AI讨论
```

## 7. 常见问题

### Q1：任务执行失败怎么办？

**A**：进入任务详情页查看错误日志，根据错误信息调整需求描述，重新创建任务。

### Q2：如何修改已创建的任务？

**A**：目前任务创建后不支持修改，如需调整请创建新任务。

### Q3：如何删除任务？

**A**：在任务列表页找到对应任务，点击删除按钮（如有）。

### Q4：会话可以导出吗？

**A**：目前不支持导出，后续版本将增加此功能。
