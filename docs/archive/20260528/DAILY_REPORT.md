1. 文档核心用途
每日09:00由AI流水线自动汇总生成，整合夜间所有执行轮次的成果、问题、待办事项，供人工快速查阅，无需逐轮核对细节；核心作用是“复盘夜间成果+明确日间人工操作项”，贴合无人值守闭环的晨间交付需求。
2. 规范要求
- 每日09:00自动生成，覆盖前一日22:00至当日09:00的所有执行轮次，禁止遗漏轮次数据；
- 成果需量化，问题需分类，待审批项需单独列出，方便人工快速决策（仅需回复APPROVE/REJECT）；
- 需包含“组件复用情况”“规范合规情况”，体现执行层的核心要求；
- 结尾需明确“次日夜间重点推进方向”，衔接下一轮流水线执行。
3. 标准模板
