# AutoCodeForge 后端开发技能需求清单

**文档版本**: v3.0
**生成日期**: 2026-05-19
**适用对象**: 后端开发团队

---

## 一、基础必备打底技能（入门必会）

| 技能项 | 掌握深度 | 核心要点 | 为什么必需 |
|--------|---------|---------|-----------|
| C# 语法功底 | 熟练 | 泛型、扩展方法、异步 async/await、委托、特性、枚举、JSON 序列化 | 项目大量使用异步 IO、特性驱动的 ORM（SqlSugar attribute）、JSON 存储与序列化 |
| ASP.NET Core 核心 | 熟练 | 依赖注入 DI、中间件、生命周期、配置读取、多环境配置 | Minimal API + 中间件/DI 是项目骨干，认证、全局异常、日志都通过中间件与 DI 注入实现 |
| Minimal API 专精 | 熟练 | 路由定义、参数绑定、分组接口、REST 规范、接口版本控制 | 项目采用 Minimal API 作为 API 层，轻量化快速开发 |
| 数据库基础 | 入门→熟练 | SQLite 使用、表关系、基础 SQL、事务、索引设计 | 默认使用 SQLite 做 MVP 存储，需要保证事务一致性与备份策略 |

## 二、项目架构 & 工程化技能（骨架核心）

| 技能项 | 掌握深度 | 核心要点 | 为什么必需 |
|--------|---------|---------|-----------|
| 分层架构落地 | 熟练 | 基类实体→仓储→服务→接口四层分层思想，职责拆分 | 便于维护、测试与多人协作 |
| 通用封装能力 | 熟练 | 封装基类实体、通用仓储、统一返回体、全局工具类 | 减少重复代码、统一错误与响应格式 |
| 代码复用思维 | 熟练 | 公共方法抽离、扩展方法批量复用、避免重复 CRUD | 提高开发效率，保持代码一致性 |
| 项目目录规范搭建 | 入门→熟练 | 按标准目录拆分模块，多人协作统一规范 | 统一项目结构，降低沟通成本 |
| Git 协同规范 | 入门→熟练 | 分支管理、忽略配置、版本提交规范 | 保证多人协作时的代码质量 |

## 三、数据层专属技能（SqlSugar + 数据设计）

| 技能项 | 掌握深度 | 核心要点 | 为什么必需 |
|--------|---------|---------|-----------|
| SqlSugar 全套使用 | 熟练 | CodeFirst 自动建表、全局过滤器、软删除、联表查询、分页查询、批量操作 | 项目数据访问统一使用 SqlSugar，能提高开发效率与性能 |
| 实体设计规范 | 熟练 | 统一主键、时间戳、用户归属字段设计，合理控制 JSON 字段使用 | 保证数据一致性与可扩展性 |
| 数据权限控制 | 熟练 | 基于 UserId 自动数据隔离逻辑实现 | 每个用户只能看到自己的数据，保证数据安全 |

## 五、AI 核心业务技能（项目核心）

## 学习优先级建议

### MVP 第一优先级（核心必需）
1. C# 语法基础 + ASP.NET Core Minimal API
2. SqlSugar ORM + 数据库设计
3. JWT 认证 + 全局异常处理
4. Microsoft Agent Framework 基础

### MVP 第二优先级（功能必需）
5. AI 工具函数 + SSE 流式对话
6. 后台任务调度
7. Git 平台对接

### MVP 第三优先级（完善增强）
8. 通用工具封装
9. 健康检查 + Swagger
10. 运维部署
