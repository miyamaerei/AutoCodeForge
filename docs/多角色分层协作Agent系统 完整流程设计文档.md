# 多角色分层协作Agent系统 完整流程设计文档

## 一、文档概述

### 1\.1 设计背景

基于初始定时轮询Agent迭代优化，摒弃固定10分钟强制触发的低效模式，模拟企业组织协作模式，搭建**分层角色、状态可控、服务化可调用、全流程闭环**的多智能体协作系统。解决原方案任务堆叠、灵活性差、无分工、无状态管控、无法对外服务的核心问题。

### 1\.2 核心设计理念

以人为企业组织协作逻辑为核心，拆分秘书、部门老大、执行小弟三类核心Agent角色，秘书支持多实例部署，实现**统一入口、分层决策、分工执行、逐级审核、统一收口**的自动化作业闭环，同时支持外部接口调用、Agent状态动态切换、跨部门协作、异常容错重试。

### 1\.3 适用范围

适用于自动化需求处理、任务拆解、方案规划、内容产出、流程审核、结果归档等重复性、流程化、可拆解的智能作业场景，支持内部自动调度、外部系统API调用双重触发。

## 二、系统整体架构总览

### 2\.1 架构模式

分层层级架构 \+ 状态机驱动 \+ 任务队列调度 \+ 服务化接口

核心链路：**外部/内部需求触发 → 秘书Agent（入口分发） → 部门老大Agent（决策分配） → 执行小弟Agent（落地执行） → 老大审核/跨部门流转 → 秘书汇总收口 → 结果返回/归档**

### 2\.2 系统核心能力

- **角色化分工**：权责拆分，解耦可扩展，支持多部门、多秘书、多团队Agent扩容

- **状态可控**：所有Agent统一三种工作状态，杜绝任务争抢、重复执行

- **双触发模式**：支持外部API调用、内部定时/主动触发

- **全流程闭环**：需求接收\-拆解\-执行\-审核\-流转\-归档\-结果输出

- **容错自愈**：失败重试、任务转移、异常兜底、日志溯源

- **负载均衡**：基于Agent空闲状态智能分配任务，避免单点拥堵

## 三、Agent角色定义与权责规范

系统统一设置三类核心Agent角色，支持多实例部署（多部门、多小弟），所有角色独立运行、互不耦合。

### 3\.1 秘书Agent（入口/出口中枢，多实例部署）

**定位**：系统统一门户、任务调度中枢、结果收口、对外服务接口，支持**多实例部署**，多个秘书并行处理不同任务，提升系统吞吐量与容错能力

**核心权责**

- 接收外部API请求、内部定时任务、人工需求输入

- 需求合法性校验、参数校验、权限校验、限流拦截

- 任务初始化、生成唯一任务ID、记录任务起始信息

- 根据任务类型分发至对应部门老大

- 接收最终执行结果，统一整理、格式化、归档

- 对外返回标准化结果，同步任务日志与状态

- 全局任务监控，超时、异常预警

**多秘书协同规则**

- 系统初始化时预置多个秘书Agent实例，每个秘书独立运行

- 多秘书通过任务队列竞争接单，空闲秘书优先获取新任务

- 单个秘书故障不影响系统运行，其他秘书自动接管其未完成任务

- 所有秘书共享全局任务状态视图，避免重复分配

### 3\.2 部门老大Agent（决策/审核中枢）

**定位**：部门负责人，负责任务拆解、资源分配、结果审核、跨部门协作

**核心权责**

- 接收秘书分发的任务，研判任务难度、工作量、需求目标

- 拆解大任务为可执行子任务，制定执行计划

- 根据小弟Agent状态（空闲/学习/处理问题）智能分配任务

- 监控小弟执行进度，拦截卡顿、超时任务

- 审核小弟产出结果：合规性、完整性、准确性校验

- 结果不合格：打回重写/重新执行；结果合格：判定流转方向

- 支持跨部门任务流转（转交其他部门老大处理）

- 汇总部门任务执行情况，同步至秘书Agent

### 3\.3 小弟Agent（落地执行终端）

**定位**：具体任务执行者，负责需求落地、内容产出、流程执行（开发/测试/文案/分析等）

**核心权责**

- 接收部门老大下发的子任务，确认任务接收

- 落地执行具体业务：需求分析、方案撰写、文本产出、模拟开发测试等

- 实时更新自身执行状态，同步执行进度

- 任务完成后产出标准化结果，提交上级老大审核

- 执行失败时上报异常信息，等待重试或任务重分配

- 空闲时段可触发自主学习（知识库更新、技能迭代）

## 四、Agent状态机统一设计（核心关键）

所有Agent通用三种状态，状态全局可监控、可查询，任务调度严格依赖状态匹配，杜绝并发冲突、重复执行。

### 4\.1 三种状态严格定义

- **空闲状态（Idle）**：无正在处理的问题，资源释放，可随时接收新任务、新调度指令

- **学习状态（Learning）**：离线迭代能力（更新知识库、优化prompt、复盘历史任务、学习新工具），属于静默状态，**暂停接单**，不参与任务分配

- **处理问题状态（Handling）**：正在处理具体问题/任务，锁定资源，**不接收任何新任务**，直至问题处理结束/异常终止

### 4\.2 状态流转规则

空闲 → 接收任务 → 处理问题状态 → 任务完成/失败 → 空闲状态

空闲 → 主动触发学习 → 学习状态 → 学习完成 → 空闲状态

处理问题/学习状态 **禁止强制切换**，仅可自主结束后回归空闲

## 五、全局任务消息协议（标准化统一格式）

所有Agent之间的通信、任务传输、结果反馈，统一使用标准化JSON格式，杜绝上下文断裂、消息错乱。

## 六、全流程详细流转规范（核心业务流程）

### 6\.1 任务触发方式（双模式）

1. **外部触发**：第三方系统、用户通过统一API接口提交需求，支持权限校验、限流

2. **内部触发**：保留定时器能力，支持自定义周期（替代原10分钟固定定时），主动生成常态化任务

### 6\.2 完整流转步骤

**Step1：秘书Agent接收需求**

校验请求合法性、权限、参数完整性，生成唯一task\_id，初始化任务信息，标记任务状态为「等待分配」。

**Step2：秘书分发任务**

根据任务类型、业务领域，匹配对应部门，将任务推送至对应部门老大Agent。

**Step3：部门老大研判拆解**

解析需求、拆解子任务、制定执行计划，结合所有小弟Agent的实时状态，选择空闲小弟分配任务。

**Step4：小弟Agent落地执行**

小弟接收任务后，状态从「空闲」切换为「处理问题」，开始落地处理、产出对应文本/结果，实时记录执行日志。

**Step5：结果提交与老大审核**

小弟处理完成后，提交结果至部门老大，自身状态重置为「空闲」；老大对结果进行审核：

- 不通过：打回小弟重新执行，记录问题原因

- 通过：判定流转方向（内部收尾/跨部门协作）

**Step6：跨部门流转（可选）**

若任务需要其他部门协同，老大将任务流转至对应部门老大，重复上述执行审核流程。

**Step7：秘书统一收口**

所有流程处理完毕后，最终结果回流至对应秘书Agent，秘书统一格式化整理、归档全流程日志、保存产出文本。

**Step8：结果输出**

对外返回标准化任务结果、执行日志、产出内容，任务标记为「已完成」。

## 七、任务调度与负载均衡规则

### 7\.1 智能分配规则

- 仅分配任务至「空闲状态」的Agent，跳过处理问题中、学习中Agent

- 多小弟场景：轮询分配 \+ 负载均衡，优先分配任务量最少的Agent

- 高优先级任务插队，优先抢占空闲资源

### 7\.2 任务队列机制

- 所有待处理任务进入全局队列，先进先出，高优任务优先执行

- 避免瞬时大量请求导致任务丢失、拥堵堆叠

- 队列持久化，重启系统不丢失未完成任务

## 八、异常处理与容错机制（自愈能力）

### 8\.1 执行失败容错

- 小弟执行报错：自动重试2次，重试失败后上报部门老大

- 老大接收异常后，可更换空闲小弟重新执行，或人工介入标记

### 8\.2 超时熔断机制

- 设置单任务最大执行超时时间，超时自动终止任务，标记异常状态

- 避免单个卡顿任务占用Agent资源，导致整体流程阻塞

### 8\.3 单点故障规避

- 支持多部门老大部署，避免单老大单点瓶颈

- 核心任务支持备份执行、结果兜底

### 8\.4 日志溯源

全流程每一步操作、状态变更、异常信息全部记录日志，通过task\_id可完整复盘全流程。

## 九、接口服务设计

### 9\.1 对外统一入口接口

所有外部请求统一接入秘书Agent接口（多实例负载均衡），支持：任务提交、任务状态查询、结果查询、任务终止。

### 9\.2 接口安全规范

- 权限校验：Token密钥鉴权，防止非法调用

- 限流防护：限制单IP/单账号请求频次，防止接口被打爆

- 参数校验：拦截非法参数、空参数请求

## 十、系统核心优势（对比初代定时器方案）

- **资源利用率大幅提升**：无任务则空闲，不无效轮询空耗算力

- **架构更健壮**：分层解耦、角色清晰、可无限扩展多部门、多秘书、多Agent

- **状态可控可监控**：彻底解决任务重复执行、堆叠、丢失问题

- **支持商业化/产品化**：标准化接口、标准化输出、全流程可溯源

- **容错能力强**：重试、熔断、兜底、异常预警，不会单点卡死

- **适配多场景**：既支持自动化定时任务，也支持人工/外部主动调用

## 十一、可迭代扩展方向

1. 增加Agent评分机制：根据执行准确率、效率筛选优质执行Agent

2. 增加任务复盘Agent，自动总结流程问题、优化执行策略

3. 支持可视化后台：状态监控、任务流转视图、日志查询

4. 支持任务优先级自定义、黑白名单权限管控

5. 学习状态智能化：根据任务短板自动定向学习迭代
