# 故障排除

## 常见问题

### 前端问题

#### 问题 1: 页面白屏
**症状**: 访问前端显示空白页面，无任何内容

**排查步骤**:
1. **打开浏览器控制台** (F12)，查看错误信息
2. **检查网络请求**: Network 标签查看资源加载失败
3. **检查路由配置**: 确认路由匹配正确

**常见原因和解决方法**:

| 原因 | 解决方法 |
|------|---------|
| JavaScript 加载失败 | 检查 Nginx/IIS 静态文件配置 |
| 路由 404 | 确认 SPA 路由重写配置 |
| API 跨域 | 配置 CORS 策略 |

**示例解决**:
```nginx
# Nginx 配置 SPA 路由
location / {
    try_files $uri $uri/ /index.html;
}
```

---

#### 问题 2: API 请求失败 (CORS 错误)
**症状**: 控制台显示 `Access to XMLHttpRequest blocked by CORS policy`

**解决方法**:
```csharp
// 后端 Program.cs 配置 CORS
builder.Services.AddCors(options =>
{
    options.AddPolicy("AllowFrontend", policy =>
    {
        policy.WithOrigins("http://localhost:5173", "https://your-domain.com")
              .AllowAnyMethod()
              .AllowAnyHeader()
              .AllowCredentials();
    });
});

app.UseCors("AllowFrontend");
```

#### 问题 3: 登录后无法访问其他页面
**症状**: 登录成功但访问其他页面时被重定向到登录页

**排查步骤**:
1. 检查 JWT 令牌是否存储到 localStorage
2. 检查 axios 请求是否携带 Authorization 头
3. 检查后端令牌验证逻辑

**解决方法**:
```typescript
// 前端 axios 拦截器
axios.interceptors.request.use(config => {
  const token = localStorage.getItem('auth_token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})
```

#### 问题 4: npm install 失败
**症状**: `npm install` 报错或超时

**解决方法**:
```bash
# 方法 1: 清除缓存
npm cache clean --force
rm -rf node_modules package-lock.json
npm install

# 方法 2: 切换镜像
npm config set registry https://registry.npmmirror.com

# 方法 3: 使用 pnpm
npm install -g pnpm
pnpm install
```

#### 问题 5: Vite 开发服务器启动失败
**症状**: `npm run dev` 报错 `EADDRINUSE: address already in use`

**解决方法**:
```bash
# 查找占用端口的进程
lsof -i :5173  # macOS/Linux
netstat -ano | findstr :5173  # Windows

# 杀死进程
kill -9 <PID>  # macOS/Linux
taskkill /PID <PID> /F  # Windows

# 或修改端口
npm run dev -- --port 3000
```

#### 问题 7: 数据库连接失败
**症状**: 应用启动时报错 `Unable to connect to database`

**排查步骤**:
1. 检查连接字符串是否正确
2. 检查数据库服务是否运行
3. 检查防火墙和网络配置

**解决方法**:
```json
// appsettings.Development.json
{
  "ConnectionStrings": {
    // SQLite（开发环境）
    "DefaultConnection": "Data Source=autocodeforge.dev.db"
    
    // SQL Server（生产环境）
    //"DefaultConnection": "Server=localhost;Database=AutoCodeForge;User Id=sa;Password=YourPassword;TrustServerCertificate=True"
  }
}
```

**测试连接**:
```csharp
// 在 Program.cs 添加测试代码
var db = app.Services.GetRequiredService<ISqlSugarClient>();
try
{
    db.Ado.ExecuteCommand("SELECT 1");
    Console.WriteLine("Database connection successful");
}
catch (Exception ex)
{
    Console.WriteLine($"Database connection failed: {ex.Message}");
}
```

#### 问题 8: JWT 令牌验证失败
**症状**: API 请求返回 `401 Unauthorized`

**排查步骤**:
1. 检查 JWT 密钥长度（至少 32 字符）
2. 检查令牌是否过期
3. 检查 JwtAuthMiddleware 是否正确配置

**解决方法**:
```json
// appsettings.json - 确保密钥足够长
{
  "Jwt": {
    "Key": "your-secret-key-at-least-32-characters-long-for-security",
    "Issuer": "AutoCodeForge",
    "Audience": "AutoCodeForge",
    "ExpirationMinutes": 60
  }
}
```

```csharp
// Program.cs - 确保 JWT 中间件配置正确
app.UseMiddleware<JwtAuthMiddleware>();
```

#### 问题 9: SqlSugar 初始化失败

**解决方法**:

// Program.cs - 检查 SqlSugar 配置
builder.Services.AddSqlSugar(new ConnectionConfig

### 数据库问题

#### 问题 11: SQLite 数据库锁定
**症状**: `database is locked` 错误

**解决方法**:
```bash
# 关闭所有访问数据库的进程
pkill -f "AutoCodeForge.Api"

# 重新启动应用
dotnet run
```

#### 问题 12: EF Core 迁移失败
**症状**: `dotnet ef database update` 报错

**解决方法**:
```bash
# 安装 EF Core 工具
dotnet tool install --global dotnet-ef

# 检查迁移历史
dotnet ef migrations list

# 删除最后一个迁移（如果错误）
dotnet ef migrations remove

# 重新创建迁移
dotnet ef migrations add InitialCreate

# 应用迁移
dotnet ef database update
```

#### 问题 15: HTTPS 证书错误
**症状**: 浏览器显示 "不安全连接" 或证书错误

**解决方法**:
```bash
# 检查证书有效期
openssl x509 -in /etc/letsencrypt/live/your-domain.com/cert.pem -text -noout

# 续期证书
sudo certbot renew

# 重载 Nginx
sudo systemctl reload nginx
```

## 性能问题

### 问题 16: 前端加载慢
**症状**: 首屏加载时间超过 3 秒

**优化方法**:
1. **启用 Gzip 压缩**:
   ```nginx
   gzip on;
   gzip_types text/plain text/css application/json application/javascript;
   ```

2. **代码分割**:
   ```typescript
   // 路由懒加载
   const TaskCenter = () => import('./modules/task-center/views/TaskList.vue')
   ```

3. **静态资源 CDN**:
   ```html
   <!-- 使用 CDN 加载库 -->
   <script src="https://cdn.jsdelivr.net/npm/vue@3.5.32"></script>
   ```

### 问题 17: 后端响应慢
**症状**: API 请求响应时间超过 1 秒

**排查步骤**:
1. **启用 SQL 日志**: 查看慢查询
2. **添加数据库索引**: 优化查询性能
3. **启用缓存**: Redis 缓存热点数据

**优化示例**:
```csharp
// 添加索引
public class Task
{
    [SugarColumn(IsIndex = true)]
    public string Status { get; set; }
    
    [SugarColumn(IsIndex = true)]
    public DateTime CreatedAt { get; set; }
}

// 使用缓存
public async Task<TaskDto> GetTaskByIdAsync(int taskId)
{
    var cacheKey = $"task:{taskId}";
    var cached = await _cache.GetAsync<TaskDto>(cacheKey);
    if (cached != null) return cached;
    
    var task = await _repository.GetByIdAsync(taskId);
    await _cache.SetAsync(cacheKey, task, TimeSpan.FromMinutes(5));
    return task;
}
```

## 错误码参考

| 错误码 | HTTP 状态 | 说明 | 解决方法 |
|--------|-----------|------|----------|
| `AUTH_001` | 401 | 令牌无效 | 重新登录获取令牌 |
| `AUTH_002` | 401 | 令牌过期 | 使用 Refresh Token 刷新 |
| `DB_001` | 500 | 数据库连接失败 | 检查连接字符串和数据库服务 |
| `DB_002` | 500 | 查询超时 | 优化查询或增加超时时间 |
| `VAL_001` | 400 | 参数验证失败 | 检查请求参数格式 |
| `NOT_FOUND` | 404 | 资源不存在 | 确认资源 ID 正确 |
