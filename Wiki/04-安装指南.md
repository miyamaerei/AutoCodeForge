# 安装指南

## 环境准备

### 必备软件

#### 1. Node.js
**版本要求**: `^20.19.0` 或 `>=22.12.0`

**安装方法**:
- **Windows**: 从 [Node.js 官网](https://nodejs.org/) 下载安装包
- **macOS**: 
  ```bash
  brew install node@20
  ```
- **Linux**:
  ```bash
  curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
  sudo apt-get install -y nodejs
  ```

**验证安装**:
```bash
node --version  # 应显示 v20.x.x 或 v22.x.x
npm --version   # 应显示 8.x.x 或更高
```

---

#### 2. .NET SDK
**版本要求**: `10.0` 或更高

**安装方法**:
- **Windows**: 从 [.NET 官网](https://dotnet.microsoft.com/download) 下载安装包
- **macOS**:
  ```bash
  brew install --cask dotnet-sdk
  ```
- **Linux**:
  ```bash
  wget https://dot.net/v1/dotnet-install.sh
  chmod +x dotnet-install.sh
  ./dotnet-install.sh --channel 10.0
  ```

**验证安装**:
```bash
dotnet --version  # 应显示 10.x.x
```

---

#### 3. Git
**版本要求**: `>=2.30`

**安装方法**:
- **Windows**: 从 [Git 官网](https://git-scm.com/) 下载安装包
- **macOS**: `brew install git`
- **Linux**: `sudo apt-get install git`

**验证安装**:
```bash
git --version  # 应显示 2.x.x
```

---

### 可选软件

#### 4. Visual Studio Code
推荐的 IDE，支持前后端开发。

**安装扩展**:
- Vue - Official
- C# Dev Kit
- Prettier - Code formatter
- ESLint
- GitLens

#### 5. SQLite Browser
查看和编辑 SQLite 数据库。

**下载**: https://sqlitebrowser.org/

## 安装前端依赖

```bash
# 进入前端目录
cd client

# 安装依赖（使用 npm）
npm install

# 或使用 pnpm（更快）
pnpm install
```

**常见问题**:
- **问题**: `npm install` 速度慢或失败
- **解决**: 切换 npm 镜像
  ```bash
  npm config set registry https://registry.npmmirror.com
  ```

---

## 安装后端依赖

```bash
# 返回项目根目录
cd ..

# 进入后端目录
cd server

# 恢复 NuGet 包
dotnet restore
```

**常见问题**:
- **问题**: `dotnet restore` 超时
- **解决**: 配置 NuGet 镜像
  ```bash
  dotnet nuget add source https://mirrors.aliyun.com/nuget/v3/index.json -n aliyun
  ```

## 配置环境变量

### 后端配置

1. **复制配置模板**:
   ```bash
   cd src/AutoCodeForge.Api
   cp appsettings.json appsettings.Development.json
   ```

2. **编辑 `appsettings.Development.json`**，设置以下关键配置：

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Data Source=autocodeforge.dev.db"
  },
  "Jwt": {
    "Key": "your-secret-key-at-least-32-characters-long",
    "Issuer": "AutoCodeForge",
    "Audience": "AutoCodeForge",
    "ExpirationMinutes": 60
  },
  "Encryption": {
    "Key": "your-base64-encryption-key-32-bytes"
  },
  "Git": {
    "GitHub": {
      "ClientId": "your-github-client-id",
      "ClientSecret": "your-github-client-secret"
    },
    "GitLab": {
      "ClientId": "your-gitlab-client-id",
      "ClientSecret": "your-gitlab-client-secret"
    },
    "AzureDevOps": {
      "ClientId": "your-azuredevops-client-id",
      "ClientSecret": "your-azuredevops-client-secret"
    }
  }
}
```

**生成密钥**:
```bash
# 生成 JWT 密钥（至少 32 字符）
openssl rand -base64 32

# 生成加密密钥（32 字节）
openssl rand -base64 32
```

## 初始化数据库

AutoCodeForge 使用 SQLite 作为开发数据库，首次运行时自动初始化。

```bash
# 确保在后端 API 目录
cd server/src/AutoCodeForge.Api

# 运行后端（首次运行自动创建数据库）
dotnet run
```

**验证**:
- 应在当前目录生成 `autocodeforge.dev.db` 文件
- 控制台输出 "Database initialized successfully"（如果有）

**使用 SQLite Browser 查看**:
1. 打开 SQLite Browser
2. 打开文件: `server/src/AutoCodeForge.Api/autocodeforge.dev.db`
3. 查看表结构

## 启动应用

### 启动后端

```bash
# 在 server/src/AutoCodeForge.Api 目录
dotnet run

# 或使用 watch 模式（文件变更自动重启）
dotnet watch run
```

**验证**:
- 访问 Swagger UI: http://localhost:5000/swagger
- 访问健康检查: http://localhost:5000/api/health

**默认端口**: `5000`（HTTP）、`5001`（HTTPS）

---

### 启动前端

```bash
# 在 client 目录
npm run dev

# 或使用 pnpm
pnpm dev
```

**验证**:
- 访问前端: http://localhost:5173
- 应显示登录页面

**默认端口**: `5173`

## 首次登录

### 创建管理员账户

**方法 1: 使用 Swagger UI**
1. 访问 http://localhost:5000/swagger
2. 找到 `POST /api/auth/register` 端点
3. 执行注册请求:
   ```json
   {
     "username": "admin",
     "password": "Admin@123",
     "email": "admin@example.com"
   }
   ```

**方法 2: 使用 curl**
```bash
curl -X POST http://localhost:5000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "Admin@123",
    "email": "admin@example.com"
  }'
```

### 登录系统
1. 访问前端: http://localhost:5173
2. 输入用户名和密码
3. 登录成功后进入控制台

## 常见安装问题

### 问题 1: Node.js 版本不匹配
**错误信息**: `The engine "node" is incompatible with this module`

**解决方法**:
```bash
# 安装 nvm（Node Version Manager）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 安装正确版本
nvm install 20
nvm use 20
```

---

### 问题 2: .NET SDK 未找到
**错误信息**: `dotnet: command not found`

**解决方法**:
- **Windows**: 重新安装 .NET SDK，确保勾选"添加到 PATH"
- **Linux/macOS**: 添加到环境变量
  ```bash
  echo 'export PATH=$PATH:$HOME/.dotnet' >> ~/.bashrc
  source ~/.bashrc
  ```

---

### 问题 3: 端口冲突
**错误信息**: `Address already in use: 5000`

**解决方法**:
```bash
# 查找占用端口的进程
lsof -i :5000  # macOS/Linux
netstat -ano | findstr :5000  # Windows

# 杀死进程
kill -9 <PID>  # macOS/Linux
taskkill /PID <PID> /F  # Windows

# 或修改端口
# 编辑 server/src/AutoCodeForge.Api/Properties/launchSettings.json
```

---

### 问题 4: SQLite 数据库权限错误
**错误信息**: `unable to open database file`

**解决方法**:
```bash
# 检查目录权限
chmod 755 server/src/AutoCodeForge.Api

# 检查数据库文件权限
chmod 644 server/src/AutoCodeForge.Api/autocodeforge.dev.db
```

---

### 问题 5: JWT 密钥长度不足
**错误信息**: `JWT key must be at least 32 characters`

**解决方法**:
```bash
# 生成新密钥（至少 32 字符）
openssl rand -base64 32

# 更新 appsettings.Development.json 中的 Jwt.Key
```

---

### 问题 6: npm install 失败（网络问题）
**错误信息**: `ETIMEDOUT` 或 `ECONNREFUSED`

**解决方法**:
```bash
# 方法 1: 切换 npm 镜像
npm config set registry https://registry.npmmirror.com

# 方法 2: 使用代理
npm config set proxy http://proxy.example.com:8080
npm config set https-proxy http://proxy.example.com:8080

# 方法 3: 使用 pnpm（更稳定）
npm install -g pnpm
pnpm install
```
