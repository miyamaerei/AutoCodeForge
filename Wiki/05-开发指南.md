# 开发指南

## 开发流程

### 标准开发流程
```
1. 创建 Issue/任务
   ↓
2. 创建功能分支
   ↓
3. 本地开发
   ↓
4. 编写测试
   ↓
5. 提交代码
   ↓
6. 创建 Pull Request
   ↓
7. 代码审查
   ↓
8. 合并到主分支
```

---

## 分支策略

### 分支类型

| 分支类型 | 命名规则 | 用途 | 示例 |
|---------|---------|------|------|
| **main** | `main` | 稳定主分支，受保护 | `main` |
| **feature** | `feature/模块/功能名` | 新功能开发 | `feature/task-center/priority-field` |
| **bugfix** | `bugfix/模块/问题描述` | Bug 修复 | `bugfix/auth/token-refresh` |
| **hotfix** | `hotfix/问题描述` | 紧急修复 | `hotfix/security-patch` |
| **refactor** | `refactor/模块/重构内容` | 代码重构 | `refactor/api/error-handling` |
| **docs** | `docs/文档类型` | 文档更新 | `docs/api-guide` |

---

### 创建功能分支

```bash
# 从最新的 main 分支创建
git checkout main
git pull origin main
git checkout -b feature/task-center/priority-field

# 开发...

# 定期同步 main 分支
git fetch origin main
git rebase origin/main
```

## 代码规范

### 后端代码规范 (C#)

#### 命名规范

| 类型 | 命名规则 | 示例 |
|------|---------|------|
| **类名** | PascalCase | `TaskService`, `BaseRepository` |
| **接口名** | I + PascalCase | `ITaskService`, `IBaseRepository` |
| **方法名** | PascalCase | `GetTaskByIdAsync`, `CreateTask` |
| **参数/变量** | camelCase | `taskId`, `userName` |
| **常量** | PascalCase | `MaxRetryCount`, `DefaultTimeout` |
| **私有字段** | _camelCase | `_taskRepository`, `_logger` |

#### 异步方法规范
```csharp
// ✅ 推荐：异步方法以 Async 结尾
public async Task<TaskDto> GetTaskByIdAsync(int taskId)
{
    return await _taskRepository.GetByIdAsync(taskId);
}

// ❌ 避免：异步方法不加 Async 后缀
public async Task<TaskDto> GetTaskById(int taskId)
{
    return await _taskRepository.GetByIdAsync(taskId);
}
```

#### 注释规范
```csharp
/// <summary>
/// 根据任务 ID 获取任务详情
/// </summary>
/// <param name="taskId">任务 ID</param>
/// <returns>任务 DTO</returns>
/// <exception cref="NotFoundException">任务不存在时抛出</exception>
public async Task<TaskDto> GetTaskByIdAsync(int taskId)
{
    // 业务逻辑注释
    var task = await _taskRepository.GetByIdAsync(taskId);
    if (task == null)
    {
        throw new NotFoundException($"Task with ID {taskId} not found");
    }
    return task.ToDto();
}
```

#### 仓储继承规范
**所有仓储必须继承 `BaseRepository<T>`**:
```csharp
public class TaskRepository : BaseRepository<Task>, ITaskRepository
{
    public TaskRepository(ISqlSugarClient db) : base(db)
    {
    }
    
    // 额外方法...
}
```

### 前端代码规范 (TypeScript)

#### 命名规范

| 类型 | 命名规则 | 示例 |
|------|---------|------|
| **组件名** | PascalCase | `TaskList.vue`, `TaskForm.vue` |
| **文件名** | kebab-case | `task-list.vue`, `use-task.ts` |
| **函数名** | camelCase | `fetchTasks`, `handleSubmit` |
| **变量名** | camelCase | `taskId`, `isLoading` |
| **常量** | UPPER_SNAKE_CASE | `MAX_RETRY_COUNT`, `API_BASE_URL` |
| **类型/接口** | PascalCase | `TaskDto`, `ApiResponse` |

#### Pinia Store 规范
**所有 Store 使用 Setup Store 格式**:
```typescript
// ✅ 推荐：Setup Store
export const useTaskStore = defineStore('task', () => {
  const tasks = ref<Task[]>([])
  const loading = ref(false)
  
  const fetchTasks = async () => {
    loading.value = true
    try {
      const response = await taskApi.getTasks()
      tasks.value = response.data
    } finally {
      loading.value = false
    }
  }
  
  return { tasks, loading, fetchTasks }
})

// ❌ 避免：Options Store
export const useTaskStore = defineStore('task', {
  state: () => ({
    tasks: [],
    loading: false
  }),
  actions: {
    async fetchTasks() {
      // ...
    }
  }
})
```

#### 组件规范
```vue
<script setup lang="ts">
// 1. 导入
import { ref, computed, onMounted } from 'vue'

</script>

<template>

    <!-- 模板内容 -->
  </div>
</template>

<style scoped>

  /* 样式 */

```

## 提交规范

### Conventional Commits

遵循 [Conventional Commits](https://www.conventionalcommits.org/) 规范。

**格式**:
```
<类型>(<范围>): <简短描述>

[可选的详细描述]

[可选的 Footer]
```

**提交类型**:
| 类型 | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat(task-center): 添加任务优先级字段` |
| `fix` | Bug 修复 | `fix(auth): 修复登录令牌刷新逻辑` |
| `docs` | 文档更新 | `docs(wiki): 更新安装指南` |
| `style` | 代码格式（不影响功能） | `style(api): 格式化代码` |
| `refactor` | 代码重构 | `refactor(task): 重构任务服务层` |
| `test` | 测试相关 | `test(task): 新增任务优先级单元测试` |
| `chore` | 构建/工具相关 | `chore(deps): 升级依赖版本` |
| `perf` | 性能优化 | `perf(api): 优化任务查询性能` |
| `ci` | CI/CD 相关 | `ci(github): 添加自动部署工作流` |

**示例**:
```bash
# 简单提交
git commit -m "feat(task-center): 添加任务优先级字段"

# 详细提交
git commit -m "fix(auth): 修复登录令牌刷新逻辑

令牌过期后无法正常刷新，导致用户频繁退出登录。
修复逻辑：
- 在令牌过期前 5 分钟自动刷新
- 刷新失败时清除本地令牌并重定向到登录页

Closes #123"
```

3. 查看表结构和数据
4. 执行 SQL 查询

#### 使用 SqlSugar 日志
在 `Program.cs` 中启用 SQL 日志:
```csharp
builder.Services.AddSqlSugar(new ConnectionConfig
{
    ConnectionString = connectionString,
    DbType = DbType.Sqlite,
    IsAutoCloseConnection = true,
    InitKeyType = InitKeyType.Attribute,
    
    // 启用 SQL 日志
    ConfigureExternalServices = new ConfigureExternalServices
    {
        DataInfoCacheService = new SqlSugarCache()
    },
    MoreSettings = new ConnMoreSettings
    {
        IsAutoRemoveDataCache = true,
        SqlServerCodeFirstNvarchar = true
    }
}, db =>
{
    // 打印 SQL 日志到控制台
    db.Aop.OnLogExecuting = (sql, pars) =>
    {
        Console.WriteLine($"SQL: {sql}");
        Console.WriteLine($"Parameters: {string.Join(", ", pars.Select(p => $"{p.ParameterName}={p.Value}"))}");
    };
});
```

// TaskServiceTests.cs
public class TaskServiceTests
{
    private readonly ITaskService _taskService;
    private readonly Mock<ITaskRepository> _mockRepository;
    
    public TaskServiceTests()
    {
        _mockRepository = new Mock<ITaskRepository>();
        _taskService = new TaskService(_mockRepository.Object);
    }
    
    [Fact]
    public async Task GetTaskByIdAsync_ShouldReturnTask_WhenTaskExists()
    {
        // Arrange
        var taskId = 1;
        var task = new Task { Id = taskId, Title = "Test Task" };
        _mockRepository.Setup(r => r.GetByIdAsync(taskId)).ReturnsAsync(task);
        
        // Act
        var result = await _taskService.GetTaskByIdAsync(taskId);
        
        // Assert
        Assert.NotNull(result);
        Assert.Equal(taskId, result.Id);
    }
}
