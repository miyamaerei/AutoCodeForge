## 版本规范

### 版本号格式

遵循 [Semantic Versioning 2.0.0](https://semver.org/) 规范：
```
主版本号.次版本号.修订号[-预发布标识]
```

**示例**:
- `1.0.0` - 稳定版本
- `1.1.0` - 新增功能（向后兼容）
- `1.0.1` - Bug 修复（向后兼容）
- `2.0.0` - 破坏性变更（不向后兼容）
- `1.0.0-alpha` - Alpha 版本
- `1.0.0-beta` - Beta 版本
- `1.0.0-rc.1` - Release Candidate

---

### 变更类型

| 类型 | 说明 | 版本号变更 |
|------|------|-----------|
| **破坏性变更** | API 不兼容变更 | 主版本号 +1 |
| **新功能** | 向后兼容的新功能 | 次版本号 +1 |
| **Bug 修复** | 向后兼容的 Bug 修复 | 修订号 +1 |
| **文档更新** | 文档变更 | 无版本号变更 |
| **性能优化** | 性能改进 | 修订号 +1 |

## 发布说明

### 如何发布新版本

1. **更新版本号**:
   ```bash
   # 前端
   cd client
   npm version major|minor|patch
   
   # 后端
   # 编辑 *.csproj 文件中的 <Version>
   ```

2. **更新 CHANGELOG**:
   - 在本文件顶部添加新版本条目
   - 记录所有变更

3. **创建 Git Tag**:
   ```bash
   git tag -a v1.0.0 -m "Release v1.0.0"
   git push origin v1.0.0
   ```

4. **创建 GitHub Release**:
   - 访问 https://github.com/miyamaerei/AutoCodeForge/releases
   - 创建新 Release
   - 复制 CHANGELOG 内容到 Release Notes

5. **部署到生产环境**:
   - 参见 [部署指南](07-部署指南.md)
