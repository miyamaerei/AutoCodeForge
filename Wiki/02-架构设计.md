# 架构设计

## 整体架构

AutoCodeForge 采用**前后端分离**架构，前端使用 Vue 3 构建 SPA 应用，后端使用 .NET 10.0 构建 RESTful API 服务。

```
┌─────────────────────────────────────────────────────────────┐
│                         用户界面                              │
│                     Vue 3 + Element Plus                     │
└─────────────────────────────────────────────────────────────┘
                              │
                    HTTP REST API + JWT
                              │
┌─────────────────────────────────────────────────────────────┐
│                     后端 API 层                              │
│                  .NET 10.0 Minimal API                      │
├─────────────────────────────────────────────────────────────┤
│                   应用层（业务逻辑）                           │
│      AI Service | Task Service | Config Service            │
├─────────────────────────────────────────────────────────────┤
│                   核心层（实体与接口）                         │
│         Entities | DTOs | Interfaces                        │
├─────────────────────────────────────────────────────────────┤
│                 基础设施层（数据与集成）                        │
│    SqlSugar ORM | Git Integration | External APIs          │
└─────────────────────────────────────────────────────────────┘
                              │
                    SQLite / SQL Server
                              │
                        持久化存储
```

## 后端四层架构

### Api 层 (AutoCodeForge.Api)
**职责**: HTTP 端点、中间件、启动配置

**核心组件**:
- **Endpoints/**: 23 个端点类，使用 Minimal API 风格
  - `AuthEndpoints.cs` - 认证和授权
  - `TaskEndpoints.cs` - 任务管理
  - `AgentEndpoints.cs` - Agent 管理
  - `ChatEndpoints.cs` - 聊天对话
  - 等等...
- **Middleware/**: 自定义中间件
  - `JwtAuthMiddleware.cs` - JWT 认证中间件
  - `ExceptionHandlingMiddleware.cs` - 全局异常处理
- **Program.cs**: 应用启动入口
  - 服务注册（DI）
  - 中间件配置
  - CORS 策略
  - Swagger 集成

**证据**: `server/src/AutoCodeForge.Api/`

---

### Application 层 (AutoCodeForge.Application)
**职责**: 业务服务、状态机、AI 集成

**核心模块**:
- **AI/**: AI 服务
  - `LLMGatewayService.cs` - LLM 网关服务
  - `AgentExecutor.cs` - Agent 执行引擎
- **Services/**: 业务服务
  - `TaskService.cs` - 任务业务逻辑
  - `AgentService.cs` - Agent 业务逻辑
  - `ConfigService.cs` - 配置服务
- **StateMachines/**: 状态机
  - `TaskStateMachine.cs` - 任务状态转换
  - `PipelineStateMachine.cs` - 流水线状态转换
- **Security/**: 安全服务
  - `JwtTokenService.cs` - JWT 令牌生成和验证
  - `EncryptionService.cs` - 数据加密解密

**证据**: `server/src/AutoCodeForge.Application/`

---

### Core 层 (AutoCodeForge.Core)
**职责**: 实体、DTO、接口定义

**核心内容**:
- **Entities/**: 数据库实体
  - `Task.cs` - 任务实体
  - `Agent.cs` - Agent 实体
  - `Pipeline.cs` - 流水线实体
  - `Repository.cs` - 仓库实体
- **DTOs/**: 数据传输对象
  - `TaskDto.cs` - 任务 DTO
  - `AgentDto.cs` - Agent DTO
- **Interfaces/**: 服务和仓储接口
  - `ITaskService.cs` - 任务服务接口
  - `ITaskRepository.cs` - 任务仓储接口

**证据**: `server/src/AutoCodeForge.Core/`

---

### Infrastructure 层 (AutoCodeForge.Infrastructure)
**职责**: 数据访问、外部集成

**核心模块**:
- **Repositories/**: 仓储实现
  - `BaseRepository.cs` - 基础仓储（CRUD）
  - `TaskRepository.cs` - 任务仓储
  - `AgentRepository.cs` - Agent 仓储
- **Git/**: Git 集成
  - `GitHubService.cs` - GitHub API 集成
  - `GitLabService.cs` - GitLab API 集成
  - `AzureDevOpsService.cs` - Azure DevOps API 集成
- **External/**: 外部 API 集成
  - HTTP 客户端封装
  - 第三方服务集成

**证据**: `server/src/AutoCodeForge.Infrastructure/`

## 前端模块架构

### 模块优先结构 (Module-First)

每个功能模块独立封装，包含 API、类型、Store、路由和视图。

```
client/src/modules/
├── auth/                    # 认证模块
│   ├── index.ts            # 模块导出
│   ├── routes.ts           # 路由定义
│   ├── auth.api.ts         # 认证 API
│   ├── auth.types.ts       # 认证类型
│   └── views/              # 登录/注册页面
├── task-center/            # 任务中心模块
│   ├── index.ts
│   ├── routes.ts
│   ├── task.api.ts
│   ├── task.types.ts
│   ├── store/              # Pinia Store
│   │   └── task.store.ts
│   └── views/
├── agent-center/           # Agent 管理模块
│   ├── ...
└── [其他 10 个模块]
```

**优势**:
- ✅ **高内聚**: 模块内功能完整独立
- ✅ **低耦合**: 模块间依赖最小化
- ✅ **易维护**: 功能变更影响范围可控
- ✅ **可扩展**: 新增功能只需新增模块

**证据**: `client/src/modules/`

---

### 状态管理 (Pinia)

使用 **Pinia Setup Store** 格式，支持 TypeScript 类型推断。

**全局 Store**:
- `stores/auth.store.ts` - 认证状态（用户信息、令牌）
- `stores/theme.store.ts` - 主题配置（待确认）

**模块 Store**:
- `modules/task-center/store/task.store.ts` - 任务状态
- `modules/agent-center/store/agent.store.ts` - Agent 状态

**证据**: `client/src/stores/`、各模块 `store/` 目录

---

### 路由管理 (Vue Router)

**路由聚合**: `client/src/router/index.ts` 聚合所有模块路由

**路由守卫**:
- **全局前置守卫**: 检查 JWT 令牌
- **路由元信息**: `meta.requiresAuth` 标记需要认证的路由
- **重定向**: 未认证用户重定向到 `/login`

**懒加载**: 所有路由组件使用动态 `import()` 懒加载

**证据**: `client/src/router/index.ts#L31-L48`

## 通信方式

### 前后端通信
- **协议**: HTTP/HTTPS REST API
- **认证**: JWT Bearer Token
- **数据格式**: JSON
- **跨域**: CORS 策略配置（后端 `Program.cs` 配置）

**API 前缀**: `/api`

**JWT 令牌流程**:
1. 用户登录 → 后端验证 → 返回 JWT 令牌
2. 前端存储令牌到 `localStorage`
3. 后续请求在 `Authorization` 头中携带令牌
4. 后端 `JwtAuthMiddleware` 验证令牌
5. 令牌过期 → 使用 Refresh Token 刷新

## 数据流向

### 请求流程（前端 → 后端）
```
用户操作
  ↓
Vue 组件
  ↓
Pinia Store (可选)
  ↓
API 调用 (axios)
  ↓
后端 API 层（Endpoints）
  ↓
应用层（Services）
  ↓
基础设施层（Repositories）
  ↓
数据库（SQLite/SQL Server）
```

### 响应流程（后端 → 前端）
```
数据库查询结果
  ↓
Repository 返回实体
  ↓
Service 转换为 DTO
  ↓
Endpoint 返回 JSON
  ↓
Axios 接收响应
  ↓
Pinia Store 更新状态
  ↓
Vue 组件响应式更新
```

## 安全设计

- ✅ **JWT 认证**: 无状态认证，支持令牌刷新
- ✅ **密码加密**: BCrypt 或类似算法（待确认）
- ✅ **敏感配置加密**: `EncryptionService` 加密敏感配置
- ✅ **CORS 策略**: 限制跨域访问
- ✅ **SQL 注入防护**: ORM 参数化查询
- ✅ **XSS 防护**: 前端 Vue 自动转义
- ✅ **HTTPS**: 生产环境强制 HTTPS
