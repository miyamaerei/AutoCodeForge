## 技术选型理由

### 为什么选择 Vue 3？
- ✅ **性能优异**: Composition API + Proxy 响应式系统
- ✅ **TypeScript 友好**: 完整类型推导
- ✅ **生态成熟**: Vue Router、Pinia、Element Plus 等
- ✅ **学习曲线平缓**: 渐进式框架，易于上手

### 为什么选择 Vite？
- ✅ **开发体验**: 极快的冷启动和热更新
- ✅ **现代化**: 原生 ESM，按需编译
- ✅ **插件生态**: 丰富的插件系统
- ✅ **构建性能**: 基于 Rollup 的生产构建

### 为什么选择 Pinia？
- ✅ **官方推荐**: Vue 官方状态管理库
- ✅ **TypeScript 支持**: 完整类型推导
- ✅ **轻量简洁**: API 简单直观
- ✅ **DevTools**: 优秀的开发者工具支持

### 为什么选择 .NET 10.0？
- ✅ **高性能**: 比 Node.js 快 2-3 倍（基准测试）
- ✅ **跨平台**: Windows、Linux、macOS
- ✅ **语言特性**: C# 12 现代语言特性
- ✅ **生态完善**: NuGet 包管理器，丰富的库

### 为什么选择 Minimal API？
- ✅ **轻量**: 减少样板代码
- ✅ **高性能**: 比传统 MVC 快 30%
- ✅ **现代**: .NET 6+ 推荐方式
- ✅ **易测试**: 端点函数易于单元测试

### 为什么选择 SqlSugar？
- ✅ **性能**: 高性能 ORM
- ✅ **易用**: 流式 API，简洁直观
- ✅ **多数据库**: 支持 MySQL、SQL Server、PostgreSQL、SQLite
- ✅ **国产**: 中文文档完善

### 为什么选择 SQLite（开发环境）？
- ✅ **零配置**: 无需安装和配置
- ✅ **轻量**: 单文件数据库
- ✅ **快速**: 本地开发响应快
- ✅ **迁移**: 生产环境可无缝迁移到 SQL Server

## 前端架构模式

### Composition API
使用 Vue 3 Composition API，而非 Options API。

**优势**:
- ✅ **逻辑复用**: 可组合函数封装逻辑
- ✅ **TypeScript 支持**: 更好的类型推导
- ✅ **代码组织**: 按功能组织代码

**示例**:
```typescript
// composables/useTask.ts
export function useTask() {
  const taskStore = useTaskStore()
  const loading = ref(false)
  
  const fetchTasks = async () => {
    loading.value = true
    await taskStore.fetchTasks()
    loading.value = false
  }
  
  return { loading, fetchTasks }
}
```

### Pinia Setup Store
使用 Setup Store 格式，而非 Options Store。

**示例**:
```typescript
// stores/task.store.ts
export const useTaskStore = defineStore('task', () => {
  const tasks = ref<Task[]>([])
  
  const fetchTasks = async () => {
    const response = await taskApi.getTasks()
    tasks.value = response.data
  }
  
  return { tasks, fetchTasks }
})
```

### TypeScript 严格模式
启用 TypeScript 严格模式，提升代码质量。

**配置**: `client/tsconfig.json#L5-L10`

## 后端架构模式

### 四层架构
- **Api**: HTTP 端点
- **Application**: 业务服务
- **Core**: 实体和接口
- **Infrastructure**: 数据访问和外部集成

详见：[架构设计](02-架构设计.md)

### 依赖注入
使用 .NET 内置 DI 容器注入服务。

**示例**:
```csharp
// Program.cs
builder.Services.AddScoped<ITaskService, TaskService>();
builder.Services.AddScoped<ITaskRepository, TaskRepository>();
```

### 仓储模式
数据访问层使用仓储模式，抽象数据库操作。

**基础仓储**:
```csharp
public class BaseRepository<T> : IBaseRepository<T> where T : class, new()
{
    protected readonly ISqlSugarClient _db;
    
    public async Task<T> GetByIdAsync(int id)
    {
        return await _db.Queryable<T>().InSingleAsync(id);
    }
    
    // 其他 CRUD 方法...
}
```
