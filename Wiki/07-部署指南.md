# 部署指南

## 部署架构

### 推荐部署架构

```
                       Internet
                          │
                    [Load Balancer]
                          │
            ┌─────────────┴─────────────┐
            │                           │
      [Frontend Server]           [Backend Server]
      (Nginx + Vue SPA)           (.NET 10.0 API)
            │                           │
            └─────────────┬─────────────┘
                          │
                    [SQL Server]
```

## 环境要求

### 生产环境

#### 服务器配置
- **CPU**: 2 核或更多
- **内存**: 4 GB 或更多
- **磁盘**: 20 GB 或更多
- **操作系统**: Linux (Ubuntu 22.04+), Windows Server 2019+

#### 软件依赖
- **前端**: Nginx 1.20+ 或 IIS 10+
- **后端**: .NET 10.0 Runtime
- **数据库**: SQL Server 2019+ 或 PostgreSQL 13+
- **可选**: Docker 20.10+, Kubernetes 1.24+

## 数据库迁移

### 从 SQLite 迁移到 SQL Server

#### 步骤 1: 导出 SQLite 数据
```bash
# 使用 sqlite3 命令行工具
sqlite3 autocodeforge.dev.db .dump > backup.sql
```

#### 步骤 2: 创建 SQL Server 数据库
```sql
CREATE DATABASE AutoCodeForge;
GO

USE AutoCodeForge;
GO
```

#### 步骤 3: 更新连接字符串
编辑 `appsettings.Production.json`:
```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=sql-server-host;Database=AutoCodeForge;User Id=sa;Password=YourPassword;TrustServerCertificate=True"
  }
}
```

#### 步骤 4: 运行 EF Core 迁移
```bash
cd server/src/AutoCodeForge.Api

# 添加迁移
dotnet ef migrations add InitialCreate

# 应用迁移
dotnet ef database update
```

## 前端部署

### 方式 1: Nginx 部署

#### 步骤 1: 构建前端
```bash
cd client

# 安装依赖
npm install

# 构建生产版本
npm run build

# 构建产物在 dist/ 目录
```

#### 步骤 2: 配置 Nginx
创建 Nginx 配置文件 `/etc/nginx/sites-available/autocodeforge`:
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    root /var/www/autocodeforge;
    index index.html;
    
    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
    
    # SPA 路由处理
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # API 代理
    location /api/ {
        proxy_pass http://backend-server:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
    
    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
}
```

#### 步骤 3: 部署文件
```bash
# 复制构建产物到 Nginx 根目录
sudo mkdir -p /var/www/autocodeforge
sudo cp -r client/dist/* /var/www/autocodeforge/

# 设置权限
sudo chown -R www-data:www-data /var/www/autocodeforge
```

#### 步骤 4: 启用站点
```bash
# 启用站点配置
sudo ln -s /etc/nginx/sites-available/autocodeforge /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
```

### 方式 2: IIS 部署 (Windows)

#### 步骤 1: 构建前端
```powershell
cd client
npm install
npm run build
```

#### 步骤 2: 配置 IIS
1. 打开 IIS 管理器
2. 添加网站
   - 站点名称: AutoCodeForge
   - 物理路径: `C:\inetpub\wwwroot\autocodeforge`
   - 绑定: HTTP 端口 80

#### 步骤 3: 配置 URL Rewrite
创建 `web.config`（已包含在 `client/public/web.config`）:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
  <system.webServer>
    <rewrite>
      <rules>
        <rule name="SPA Routes" stopProcessing="true">
          <match url=".*" />
          <conditions logicalGrouping="MatchAll">
            <add input="{REQUEST_FILENAME}" matchType="IsFile" negate="true" />
            <add input="{REQUEST_FILENAME}" matchType="IsDirectory" negate="true" />
          </conditions>
          <action type="Rewrite" url="/" />
        </rule>
      </rules>
    </rewrite>
    <staticContent>
      <mimeMap fileExtension=".json" mimeType="application/json" />
    </staticContent>
  </system.webServer>
</configuration>
```

## 后端部署

### 方式 1: Systemd 服务 (Linux)

#### 步骤 1: 发布后端
```bash
cd server/src/AutoCodeForge.Api

# 发布 Release 版本
dotnet publish -c Release -o /var/www/autocodeforge-api
```

#### 步骤 2: 创建 Systemd 服务
创建 `/etc/systemd/system/autocodeforge-api.service`:
```ini
[Unit]
Description=AutoCodeForge API
After=network.target

[Service]
Type=notify
WorkingDirectory=/var/www/autocodeforge-api
ExecStart=/usr/bin/dotnet /var/www/autocodeforge-api/AutoCodeForge.Api.dll
Restart=always
RestartSec=10
KillSignal=SIGINT
SyslogIdentifier=autocodeforge-api
User=www-data
Environment=ASPNETCORE_ENVIRONMENT=Production
Environment=DOTNET_PRINT_TELEMETRY_MESSAGE=false

[Install]
WantedBy=multi-user.target
```

#### 步骤 3: 启动服务
```bash
# 重载 Systemd
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start autocodeforge-api

# 开机自启
sudo systemctl enable autocodeforge-api

# 查看状态
sudo systemctl status autocodeforge-api

# 查看日志
sudo journalctl -u autocodeforge-api -f
```

## Docker 部署

### 后端 Dockerfile

创建 `server/Dockerfile`:
```dockerfile
# 构建阶段
FROM mcr.microsoft.com/dotnet/sdk:10.0 AS build
WORKDIR /src

# 复制项目文件
COPY ["src/AutoCodeForge.Api/AutoCodeForge.Api.csproj", "src/AutoCodeForge.Api/"]
COPY ["src/AutoCodeForge.Application/AutoCodeForge.Application.csproj", "src/AutoCodeForge.Application/"]
COPY ["src/AutoCodeForge.Core/AutoCodeForge.Core.csproj", "src/AutoCodeForge.Core/"]
COPY ["src/AutoCodeForge.Infrastructure/AutoCodeForge.Infrastructure.csproj", "src/AutoCodeForge.Infrastructure/"]

# 恢复依赖
RUN dotnet restore "src/AutoCodeForge.Api/AutoCodeForge.Api.csproj"

# 复制所有源代码
COPY . .

# 构建和发布
WORKDIR "/src/src/AutoCodeForge.Api"
RUN dotnet publish -c Release -o /app/publish

# 运行时阶段
FROM mcr.microsoft.com/dotnet/aspnet:10.0 AS runtime
WORKDIR /app

# 复制构建产物
COPY --from=build /app/publish .

# 暴露端口
EXPOSE 80
EXPOSE 443

# 健康检查
HEALTHCHECK --interval=30s --timeout=3s --start-period=30s --retries=3 \
  CMD curl -f http://localhost/api/health || exit 1

# 启动应用
ENTRYPOINT ["dotnet", "AutoCodeForge.Api.dll"]
```

### 前端 Dockerfile

创建 `client/Dockerfile`:
```dockerfile
# 构建阶段
FROM node:20-alpine AS build
WORKDIR /app

# 复制依赖文件
COPY package*.json ./

# 安装依赖
RUN npm ci

# 复制源代码
COPY . .

# 构建
RUN npm run build

# 运行时阶段
FROM nginx:alpine AS runtime
WORKDIR /usr/share/nginx/html

# 删除默认文件
RUN rm -rf ./*

# 复制构建产物
COPY --from=build /app/dist .

# 复制 Nginx 配置
COPY nginx.conf /etc/nginx/conf.d/default.conf

# 暴露端口
EXPOSE 80

# 健康检查
HEALTHCHECK --interval=30s --timeout=3s CMD wget -q --spider http://localhost || exit 1

CMD ["nginx", "-g", "daemon off;"]
```

### Docker Compose

创建 `docker-compose.yml`:
```yaml
version: '3.8'

services:
  frontend:
    build:
      context: ./client
      dockerfile: Dockerfile
    ports:
      - "80:80"
    depends_on:
      - backend
    environment:
      - VITE_API_BASE_URL=http://backend:80
    networks:
      - app-network

  backend:
    build:
      context: ./server
      dockerfile: Dockerfile
    ports:
      - "5000:80"
    depends_on:
      - database
    environment:
      - ASPNETCORE_ENVIRONMENT=Production
      - ConnectionStrings__DefaultConnection=Server=database;Database=AutoCodeForge;User Id=sa;Password=YourPassword@123;TrustServerCertificate=True
      - Jwt__Key=${JWT_KEY}
      - Encryption__Key=${ENCRYPTION_KEY}
    networks:
      - app-network

  database:
    image: mcr.microsoft.com/mssql/server:2022-latest
    environment:
      - ACCEPT_EULA=Y
      - SA_PASSWORD=YourPassword@123
    ports:
      - "1433:1433"
    volumes:
      - sqlserver-data:/var/opt/mssql
    networks:
      - app-network

networks:
  app-network:
    driver: bridge

volumes:
  sqlserver-data:
```

#### 启动
```bash
# 创建 .env 文件
cat > .env << EOF
JWT_KEY=$(openssl rand -base64 32)
ENCRYPTION_KEY=$(openssl rand -base64 32)
EOF

# 启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

## 备份策略

### 数据库备份
```bash
# 每日备份脚本
#!/bin/bash
DATE=$(date +%Y%m%d)
BACKUP_DIR="/backup/database"

# 备份数据库
docker exec autocodeforge-database /opt/mssql-tools/bin/sqlcmd \
  -S localhost -U sa -P 'YourPassword@123' \
  -Q "BACKUP DATABASE [AutoCodeForge] TO DISK = N'/var/opt/mssql/backup/autocodeforge-$DATE.bak' WITH NOFORMAT, NOINIT, NAME = 'AutoCodeForge-full', SKIP, NOREWIND, NOUNLOAD, STATS = 10"

# 保留最近 30 天备份
find $BACKUP_DIR -name "*.bak" -mtime +30 -delete
```
